# V2 Release

---

## 🚩 POI Embedding

The POI semantic embedding code is located in the **`POIembedding/`** directory.  
It includes preprocessing and representation modules for constructing POI-level semantic embeddings.

---

## 🧩 SID (Semantic ID) Module

The codebook training and Semantic ID generation are provided under the **`SID/`** directory.

Key updates in V2 include:

- Adoption of **projection-based cosine-similarity quantization** instead of standard VQ
- **EMA-based smoothing** to mitigate codebook collisions during training

---

## 🦙 LLM Fine-tuning

An example implementation for LLM fine-tuning is available in the **`LLM/`** directory.
The code is adapted with reference to the following projects:
- **OneRec-Think**: <https://github.com/wangshy31/OneRec-Think/tree/main>
- **MiniOneRec**: <https://github.com/AkaliKong/MiniOneRec>

This implementation may still contain unresolved issues or incomplete components.
Therefore, we **recommend using LLaMA-Factory** for LLM training and evaluation — it is simple, efficient, and well-documented.

You may:

- First perform semantic alignment between SID and the LLM (train only the embedding module, or you can try other strategies), or
- Directly fine-tuning without alignment depending on your experimental goals.

---


## 📌 Summary

This V2 release offers an improved and modular foundation for:

- POI semantic embedding
- SID quantization and codebook learning
- LLM adaptation for generative POI recommendation
